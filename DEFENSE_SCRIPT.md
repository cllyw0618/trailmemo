# TrailMemo 三分钟答辩展示稿

老师好，我的项目叫 **TrailMemo 山野手账**，它是一款面向旅行和徒步爱好者的 3D 电子手账 Web 应用。

我做这个项目的原因是：很多人在旅行和徒步之后，照片会留在相册里，文字会分散在朋友圈、备忘录或社交平台里，记录本身比较零散，也缺少持续积累的感觉。所以我想把徒步记录做成一种更有故事感的体验，让用户像玩探索小游戏一样，点亮路线、写下记忆，并看到自己的像素旅人慢慢成长。

项目的主要流程是这样的。

首先进入启动页，用户点击“唤醒山野记忆”后，会进入认证页面。这里我设计了三种进入方式：邮箱注册、邮箱登录和游客访问。游客访问调用的是 Supabase 的匿名登录接口，会生成真实的用户 UUID，所以游客也可以保存路线和手账数据。

进入系统后，第一部分是 **3D 地图探索**。地图页使用 Three.js 渲染一个可以旋转和缩放的 3D 地球，地球上标记了国内外经典徒步路线，比如武功山、虎跳峡、雨崩、冈仁波齐、EBC、TMB 等。用户点击路线点，可以看到路线简介、难度、距离和故事提示，然后进入手账记录。

第二部分是 **电子手账系统**。用户可以选择路线，填写标题、天气、心情和文字记忆，还可以上传 1 到 4 张照片。系统会根据照片数量和模板，把内容排版成拼贴风格的电子手账。保存后，这条路线会被点亮，并写入 Supabase 数据库。

第三部分是 **成长装备系统**。这是我为了增强答辩展示效果新增的独立页面。底部导航现在有“探索、日记、成长、背包”四个入口。成长页展示像素旅人的等级变化，从 Lv.1 山野新人，到 Lv.16+ 地球漫游者。随着点亮路线数量增加，角色会逐步解锁帽子、上衣、背包、鞋子等装备，这样用户能直观看到自己的探索成长。

第四部分是 **背包记忆页面**。这里会展示用户已经点亮的路线、成就徽章、统计数据和所有手账记录。用户也可以删除不需要的手账。

技术方面，前端使用 Vue 3 和 Vite 开发，使用 Vue Router 管理页面切换，使用 Pinia 管理用户、路线、手账和角色状态。3D 地图使用 Three.js 实现，像素旅人使用 Canvas 绘制。后端和数据库使用 Supabase，包含 Auth 认证、PostgreSQL 数据库和 RLS 行级安全策略。项目最后部署到 Vercel，可以通过公网访问。

这个项目的核心亮点是：它不是一个普通的记录工具，而是把 3D 地图探索、电子手账和角色成长结合起来，让旅行记录从一次性的保存，变成一个可以长期积累的山野成长过程。

我的展示结束，谢谢老师。
